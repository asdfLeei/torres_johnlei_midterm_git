# Midterm Examination Reflection

1. What is Git used for?
 - Git is a version control system used to track changes in files over time. It allows developers to save snapshots of their project and revert to earlier versions if needed.

2. What is the difference between Git and GitHub?
 - Git is the tool that manages version control locally on a computer. GitHub is an online platform that hosts Git repositories so they can be shared and accessed remotely.

3. Why is branching useful?
 - Branching lets you work on new features or changes without affecting the main codebase. It keeps the main branch stable while allowing experimentation or parallel development.

4. What command did you use to upload your local commits to GitHub?
 - I used git push -u origin mainto upload my local commits to the GitHub repository.

5. What did you learn from this activity?
 - I learned how to set up a local Git repository, connect it to GitHub, and manage changes using branches and merges. This activity helped me understand the practical workflow developers use for version control and collaboration.
